I propose to have at least one systematic example for all implementation
as now but not in the test directory. Instead of calling it meganalysis which
is fun but confusting I suggest ACME as this is the name of the
protypical company (see wikipedia). The good thing is that if everything related
to the example is prefixed with ACME the files comes first in the directory listing
and this is handy to see what is general vs. illustration on ACME.
Below I also assume that ACMECut.bat is the program that will launch the
cut on that particular example (basically a subset of what is currently in
FeatureTests.cs)
I didn't show the directory structure here because it is not really meaningful from a
conceptual point of view when depicted like that.
Also I renamed make.bat to CompanyXSD2CS.bat to make it clearer.
Obviously we could have a big makefile, but then the corresponding rule will still
be there as Fragment.
Because of the interest making explicit the serialization, feature, deserialization,
which is at the center of the example I think that this should be in a well defined
file such as XMLFeatures.cs that wrap Features.cs.
